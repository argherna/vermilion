Vermilion

A task scheduler

Introduction

Vermilion is intended to be used as an alternative to cron on unix based systems where separate Java command line programs would be run. This is useful for jobs that are frequently run whose data updates are subject to strict time requirements. Vermilion facilitates quicker execution by having a long-running JVM process with the classes already loaded to execute your task.

Features

Tasks and Schedules are at the heart of vermilion. A Task is an implementation of a vermilion based interface, NamedRunnable, which is an extension of the familiar Runnable interface. NamedRunnables can be developed quickly and deployed to a jar file on vermilion's classpath where it will be run on its own Schedule.

Vermilion utilizes standard Java technologies at its core:

  • JMX
  • Servlets

Vermilion utilizes standard protocols for its extensions:

  • RESTful HTTP API
  • JSON

These standards make it easy for users to access information about their vermilion instances using the tools of their choice.

Configuration

Vermilion minimizes the amount of configuration work you would normally have to do by using sensible defaults. Configuration should be used for exceptional operating conditions.
